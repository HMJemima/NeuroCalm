1. Supported Inputs

The preprocessing pipeline starts by identifying what kind of file was uploaded and choosing the matching conversion path.

Model-ready CSV
  • Validate that the required 8 columns are present.
  • Convert values to numeric format.
  • Fill missing values with 0.
  • If already chunked, normalize each chunk to exactly 150 rows.
  • If not chunked, create sliding windows with window=150 and stride=3.
Raw Signal Files
  • .oxy: read time plus oxy/deoxy optode pairs.
  • .nir: read time plus 730nm, ambient, and 850nm triplets.
  • fnirSoft CSV: inspect whether the export contains hbo, hbr, oxy, raw light, time, or marker data.

2. Raw Data Conversion Paths

Each raw format is converted into a continuous signal representation before feature generation begins.

Path A: Raw .oxy

  1. Read the time column.
  2. Read oxy/deoxy optode pairs.
  3. Build oxy_matrix and deoxy_matrix.

Path B: Raw .nir

  1. Read the time column.
  2. Read 730nm, ambient, and 850nm triplets.
  3. Correct light signals using 730 - ambient and 850 - ambient.

Path C: fnirSoft CSV

  1. Inspect the export type.
  2. If it contains hbo / hbr / oxy, build oxy/deoxy matrices.
  3. If it contains raw light, apply light correction.
  4. If it contains time or markers, load timing and marker metadata.
Unify signals All raw paths feed into one continuous signal representation.
Derive proxies Build missing proxy signals when needed, including oxy, deoxy, and phase-like gradients.
Regional mapping Apply experimental mapping to turn raw channels into analysis-friendly regions.
Assumption used by the pipeline: optodes 1..8 = AB and optodes 9..16 = CD.

Scaling Mean-center, standardize, and clip to a safe range.

4. Time Alignment and Windowing

The cleaned feature channels are synchronized onto one time base, then converted into model-ready windows.

Common grid Build a common time grid at 5 Hz.
Marker trim Trim with marker times whenever markers are available.
Resample Resample all 8 features onto the same grid.
Window creation Feed the aligned features into sliding-window chunking with window=150 and stride=3.

5. Final Model Input and Output

Once chunking is complete, the data becomes ready for direct inference by the deployed SALIENT model.

Chunked dataframe Final model input is a chunked dataframe containing the chunk ID plus the 8 feature columns.
